Menus contextual e arquivos

O menu contextual

O menu de contexto para componentes de memória inclui mais quatro opções do que as comuns a todos os componentes:

Editar conteúdo :
Abrir um editor hexadecimal para editar o conteúdo de memória.
Limpar conteúdo :
Redefine as ROMs para 0 e para RAMs para 0 ou aleatoriamente, dependendo das configurações nas Opções do Projeto.
arregar imagem... :
Reinicia todos os valores na memória para os valores encontrados em um arquivo. O formato do arquivo é reconhecido automaticamente e se não for o caso, abre-se uma janela para uma seleção apropriada.
Salvar imagem... :
Armazena todos os valores na memória em um arquivo usando um dos formatos descritos abaixo.

Os formatos de arquivo

  • v2.0 raw VVersão histórica do logisim permite a codificação "run.length" Formatos de arquivo
  • v3.0 hex words plain Próximo da versão v2.0 estruturado em palavras
  • v3.0 hex words addressed Como antes, com a possibilidade de definir um endereço de início para cada linha do arquivo.
  • v3.0 hex bytes plain big-endian
    v3.0 hex bytes plain little-endian Organizado em fluxos de bytes.
  • v3.0 hex bytes addressed big-endian
    v3.0 hex bytes addressed little-endian Organizado em fluxo de bytes como antes, com a possibilidade de definir um endereço.
  • Binary data big-endian
    Binary data little-endian imagem da memória não codificada em hexadecimal.
  • Ascii byte escape big-endian
    Ascii byte escape little-endian Arquivo codificado em ASCII com código de escape, como em C para caracteres não imprimíveis.
