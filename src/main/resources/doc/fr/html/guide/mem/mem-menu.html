Le menu contextuel et les fichiers

Le menu contextuel

Le menu contextuel des composants mémoires comprend quatre options de plus que celles communes à tous les composants:

Editer le contenu :
Afficher un éditeur hexadécimal pour éditer le contenu de la mémoire.
Effacer le contenu :
Réinitialise les ROM à 0 et pour les RAM à 0 ou aléatoirement selon les paramètres dans Les options de projet.
Charger l'image ... :
Réinitialise toutes les valeurs en mémoire en fonction des valeurs trouvées dans un fichier. Le format du fichier est reconnu automatiquement et si ce n'est pas le cas une fenêtre s'ouvre pour une sélection appropriée.
Enregistrer l'image ... :
Stocke toutes les valeurs en mémoire dans un fichier en utilisant un des formats décrits ci-dessous.

Les formats de fichier

  • v2.0 raw Version historique de logisim permet un codage "run.length".
  • v3.0 hex words plain Proche de la version v2.0 et structuré en mots
  • v3.0 hex words addressed Comme précédemment avec la possibilité de définir une adresse de départ pour chaque ligne du fichier.
  • v3.0 hex bytes plain big-endian
    v3.0 hex bytes plain big-endian Organisé en flux d'octets.
  • v3.0 hex bytes addressed big-endian
    v3.0 hex bytes addressed little-endian Organisé en flux d'octets comme précédemment avec la possibilité de définir une adresse.
  • Binary data big-endian
    Binary data little-endian Image de la mémoire brute et non codée en hexadécimale.
  • Ascii byte escape big-endian
    Ascii byte escape little-endianFichier codé ASCII avec code escape, comme en C pour les caractères non imprimables.
