不足之处

Logisim 的传播算法对于几乎所有教学用途都足够精细；但对于工业电路设计而言还远远不够。从较严重到较轻微，Logisim 传播技术的不足之处包括：

  • 除了门延迟问题外，Logisim 并不特别处理时序问题。它的模型非常理想化，因此 S-R 锁存器中的一对 NOR 门会无限地同步翻转，而不是最终稳定到某个状态。

  • Logisim 无法仿真那些引脚有时作为输入、有时作为输出的子电路。不过，使用 Java 构建的组件可以具有此类引脚：在内置库中，存储器库的 RAM 电路包含一个 D 引脚，该引脚既可以作为输入，也可以作为输出。

  • Logisim 会在固定次数的迭代后终止仿真，并认为发生了振荡错误。理论上，一个并未振荡的大型电路也可能因此遇到问题。

  • Logisim 不会区分不同电压电平：一个位只能是 1、0、未知或错误。

  • 还有一些更细微的不足，本文没有展开；它们非常专业，一旦涉及这些问题，就能看出 Logisim 的模型远未达到那个层次。举一个极端例子：在大型芯片制造中，纳米级导线中的“气泡”增长并导致随机断开，也可能成为需要考虑的问题。

  • 除此之外，原作者并不是电路设计专家；因此，传播技术中也可能存在未意识到的错误。欢迎专家指正。
