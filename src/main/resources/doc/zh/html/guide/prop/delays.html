门延迟

作为 Logisim 算法复杂程度的一个例子，考虑以下电路。

这个电路“显然”总是输出 0。但实际上，非门不会对其输入立即做出反应，在 Logisim 中也是如此。因此，当该电路的输入从 0 变为 1 时，与门会短暂地看到两个 1 输入，并短暂地输出 1。屏幕上通常看不到这个瞬态变化。但如果把与门的输出用作 D 触发器的时钟输入，这种效果就可以被观察到。

用手形工具将输入从 0 切换为 1，会产生一个瞬时的 1 并送入 D 触发器。因此，每当电路输入从 0 变为 1 时，触发器的值都会翻转。借助仿真的单步模式，也可以观察到详细过程。

每个组件都有与之关联的延迟。Logisim 内置的复杂组件通常延迟更大，但这些延迟值带有一定任意性，未必反映真实硬件。


Logisim 世界


现实世界

从技术角度看，在单个电路中处理这种复杂度相对容易。跨子电路妥善处理门延迟则更复杂；Logisim 会把所有基本组件的传播事件统一放入一个调度队列中，无论这些组件位于哪个子电路。

通过项目选项窗口的仿真选项卡，可以配置 Logisim 为组件的传播添加随机的、偶发的延迟。这旨在模拟真实电路的不均匀性。特别是，使用两个或非门构建的 S-R 锁存器如果没有这种随机性就会振荡，因为两个门会同步处理其输入。此随机性默认禁用。

这并不意味着 Logisim 总能很好地处理门延迟；但至少它尝试对这类问题建模。
