提示与最佳实践

以下是一些有效使用测试向量功能的提示：

  1. 在表头中使用位宽：在表头中指定每个输入/输出引脚的位宽，即使只有1位。这对于多位值来说是必需的。
  2. 使用下划线提高可读性：您可以在数值中的任何位置使用下划线（_）。解析时会忽略下划线。示例：0x0000_1111、0o1234_5670、1111_0000、1_234 或 -5_000。
  3. 在值与表头之间使用空格对齐：对齐值和表头的起始位置可提高可读性，并更容易识别值和表头。如果不这样做，很容易混淆哪个值属于哪个表头，从而在测试失败但电路似乎工作正常时感到非常困惑。
  4. 对有状态电路使用序列：如果您的电路具有记忆功能（触发器、寄存器、计数器），请使用顺序测试来验证状态转换。
  5. 使用无关项进行部分验证：在测试复杂电路时，对当前未验证的输出使用 <DC>。
  6. 使用浮空进行三态测试：使用 <float> 来测试具有三态输出或高阻态的电路。
  7. 使用集合进行组织：使用 <set> 列将相关测试分组为序列。具有相同 <set> 编号的测试按 <seq> 顺序一起运行，步骤之间保留电路状态。
  8. 混合组合与顺序测试：您可以在同一个文件中混合组合测试（seq=0）和顺序测试。

向后兼容性

所有现有的测试向量文件无需修改即可继续使用。新功能是可选的：

  • 如果未提供 <set> 和 <seq> 列，则所有测试均为组合测试（默认行为）
  • 如果未使用特殊值 <DC> 和 <float>，则应用常规值比较
  • 原始文件格式得到完全支持
