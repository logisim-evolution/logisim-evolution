测试向量文件格式

以下是一个测试向量文件示例：

# my test vector for add32
A[8]      B[8]      C[8]      Cin Cout
0000_0000 0000_0000 0000_0000 0   0
-2        0x05      3         0   0
0         0o3       3         0   0

总体来说，文件格式很简单。表头与电路的输入/输出引脚匹配，后续各行则给出要测试的值。

第一个非空、非注释行列出每个电路输入/输出引脚的名称及其位宽（如果大于 1），以空格分隔。

初学者经常感到困惑的一点是，只要位宽不是 1 位，就必须在表头中指定。

注意：位宽不一定与值中的字符数相同。例如，一个 4 位的值 13 可以表示为 1101、00001101、0x0d、0o15、15 或 0b1101 等。

空行将被忽略。'#' 字符后的任何内容都是注释。

其余行列出每个值，以空格分隔。值可以是十六进制、八进制、二进制或有符号十进制。十六进制值必须带有 '0x' 前缀。八进制值必须带有 '0o' 前缀。二进制和十进制通过位数来区分：二进制值必须始终具有与该列位宽完全相同的位数；十进制值必须始终具有更少的位数，不应有前导零，并且可以带有负号。

为了提高可读性，可以在数值中的任何位置使用下划线（_）。解析时会忽略下划线。示例：0x0000_1111、0o1234_5670、1111_0000、1_234 或 -5_000。

对于十六进制、八进制和二进制值，数字 'x' 分别表示四个、三个或一个“不关心”位。因此，值 101xx 是一个五位二进制值，其中最后两位未指定；而 0x1ax5 是一个含有四个未指定位的十六进制值。此类“不关心”位不能用于十进制表示法。
