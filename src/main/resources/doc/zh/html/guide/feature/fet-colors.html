导线颜色

我们现在可以总结 Logisim-evolution 导线可以呈现的全部彩色颜色。 下面的小电路同时说明了所有这些。

  • 灰色： 线路的位宽未知。 发生这种情况是因为导线未连接到任何组件的输入和输出。 （所有输入和输出都有定义的位宽度。）

  • 蓝色： 线路承载一位值，但没有任何东西将特定值驱动到线路上。 我们称之为 浮动 位； 有些人称之为 高阻抗 值。 在此示例中，将值放置到线上的组件是三态引脚，因此它可以发出此浮动值。

  • 深绿色： 线路携带一位 0 值。

  • 亮绿色： 导线携带一位 1 值。

  • 黑色： 导线携带多位值。 可能未指定部分或全部位。

  • 红色： 导线带有错误值。 这种情况经常发生，因为门无法确定正确的输出，可能是因为它没有输入。 它也可能是因为两个组件试图将不同的值发送到线路上而出现的。 这就是上面示例中发生的情况，其中一个输入引脚将 0 放置到电线上，而另一个输入引脚将 1 放置到同一电线上，从而导致冲突。 当携带的任何位是错误值时，多位线将变为红色。

  • 橙色： 连接到导线的组件的位宽度不一致。 橙色线实际上是“断线”：它不在组件之间传送值。 在这里，我们将两位组件附加到一位组件上，因此它们是不兼容的。

  • 绿色-紫红色： 此配色方案表示使用工具 push ( )。 这使我们能够知道路线，并且线路状态的指示将显示在小气泡中，并且还允许遵循复杂方案中的连接。
