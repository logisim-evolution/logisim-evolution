其他验证选项

还有一些与命令行执行相关的附加选项。

--load命令行参数

更复杂的电路可能包含 RAM 或 ROM 组件，需要先加载程序或数据后电路才有可执行的内容。可以在命令行指定存储器映像文件；仿真开始前，该文件会加载到电路中的 RAM 或 ROM 组件中。（这不适用于 GUI 加载流程，只用于命令行执行。）

java -jar logisim-filename.jar cpu.circ --tty table --load ram-image.txt --load rom-image.txt myRom

--tty 的参数（见下文）必须紧跟在 --tty 之后，存储器映像文件名必须紧跟在 --load 之后。文件名后可以再跟一个存储器组件标签，表示只加载带有该标签的组件。可以根据需要使用任意多个 --load 选项。存储器映像文件应采用Logisim 的存储器映像格式。

Logisim 会递归搜索存储器，因此即使存储器嵌套在子电路中，此选项仍然有效。Logisim 会尝试把同一个文件加载到它能找到的每个标签匹配的 RAM 或 ROM 中。如果未提供标签，则同一个文件会加载到所有 RAM 或 ROM 组件中，但不会加载到与其他 --load 选项标签匹配的组件。在上面的示例中，ram-image.txt 会加载到仿真树中除标签为 myRom 的组件之外的每个 RAM 和 ROM 组件中。所有标签为 myRom 的 RAM 和 ROM 组件都会从 rom-image.txt 文件加载。--load 选项的顺序无关紧要。

--tty参数的选项

前面的示例一直使用 --tty table 表示显示输出值表。也可以列出一个或多个以逗号分隔的选项来自定义行为。例如，写作 --tty table,halt,speed 时，程序会执行下面列出的三种行为。（选项顺序无关紧要。）

halt

仿真结束后，会显示一条单行消息，解释仿真结束的原因。错误条件——例如检测到的振荡——在任何情况下都会显示。

speed

如果将 speed 与 --tty 结合使用，仿真完成后 Logisim 会显示电路仿真速度摘要，例如：

714 Hz (509 ticks in 712 milliseconds)

请注意，在仿真过程中显示信息会明显降低仿真速度。仅举一个比较：同一个电路和同一个存储器映像，仅使用 speed 选项时运行频率为 714 Hz，而同时使用 table 选项时仅为 490 Hz。

stats

显示一个以制表符分隔的表格，其中包含项目中顶层“main”电路所用组件的统计信息。该表格包含四列：

  • 唯一：该组件在电路层次结构中出现的次数，其中层次结构中的每个子电路仅计数一次。
  • 递归：该组件在电路层次结构中出现的次数，其中每个子电路在层次结构中每出现一次就计数一次。
  • 组件：组件的名称。
  • 库：组件所属库的名称。

“唯一”与“递归”之间的区别在项目菜单一节中有进一步说明。如果文件使用了已加载 Logisim 库中的电路，则这些组件被视为“黑盒”：库电路的内容不包含在唯一计数和递归计数中。

（教师要求学生只使用 Logisim 库中的某个子集完成项目时，此功能会很有用。）

table

（如前所述）

tty

所有 TTY 终端都会把输出发送到终端（标准输出），而从键盘输入的任何信息都会发送到电路中的所有键盘组件。即使这些组件深嵌在子电路层次结构中，也会被包含在内。
