VHDL 仿真器

Logisim 无法直接仿真 VHDL 内容，而是将 QuestaSim 作为后台任务调用。

启用仿真器

可以通过仿真 > 启用 VHDL 仿真菜单启用仿真器。启用后，仿真器控制台日志会显示在绘图窗格底部。日志底栏包含仿真器状态指示器，状态包括：

  • 已禁用（控制台消失）
  • 已启用（但未激活）
  • 正在启动
  • 活动

启用仿真器且电路包含 VHDL 组件时，仿真器会自动启动。如果电路中没有 VHDL 组件，则无法启动仿真器。

重新启动

重置 Logisim 仿真时，VHDL 仿真也会重新启动。这只会重置仿真状态，不会重新加载仿真进程或源文件。

如果更改了某些 VHDL 组件的内容，必须手动重新启动 VHDL 仿真器；系统不会自动完成。可以通过仿真菜单重新启动仿真器。

时间

QuestaSim 的仿真步长不可预测，因为它取决于电路中 VHDL 组件的数量。绝对最小步长为 100ns。因此，应避免使用基于时间的仿真事件（如wait for 10ns），只使用基于信号的事件。

多个实例

同一时间只能有一个 VHDL 仿真器实例。因此，如果要在第二个项目中启用它，必须先在第一个项目中禁用它。尝试在两个项目中同时启用会失败，并显示错误消息。
