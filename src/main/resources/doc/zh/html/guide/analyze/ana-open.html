打开组合分析

组合分析模块的主要功能集中在同名窗口中，可以在其中查看真值表和布尔表达式。该窗口可以通过两种方式打开。

通过窗口菜单

选择| 窗口 |→| 组合分析 |即可打开组合分析窗口。如果此前没有使用过该窗口，打开后它不会对应任何电路。

无论打开多少个项目，Logisim 中只存在一个组合分析窗口。无法同时打开两个不同的分析窗口。

通过项目菜单

在编辑电路的窗口中，也可以选择菜单| 项目 |→| 分析电路 |，让 Logisim 分析当前电路。在打开窗口之前，Logisim 会计算与电路对应的布尔表达式和真值表，并在组合分析窗口中显示它们。

要成功分析电路，每个输入必须连接到一个输入引脚，每个输出必须连接到一个输出引脚。Logisim 只会分析输入和输出各不超过八个的电路，并且所有引脚都应为一位引脚。否则会看到错误消息，窗口也不会打开。

在构建与电路对应的布尔表达式时，Logisim 首先会尝试生成与电路中各个门完全对应的表达式。但是，如果电路使用了某些不是逻辑门的组件（例如多路复用器），或者电路深度超过 100 级（这种情况并不常见），它会弹出对话框，说明无法直接推导布尔表达式。此时 Logisim 会改为先生成真值表，再根据真值表推导表达式；真值表则通过逐一尝试输入组合并读取输出结果得到。

分析完成后，原电路与组合分析窗口之间不会保持同步。也就是说，修改电路不会自动更新窗口内容；修改窗口中的布尔表达式或真值表也不会自动修改原电路。当然，随时可以再次分析电路；并且，正如后文所述，也可以用组合分析窗口当前显示内容对应的电路替换原电路。

局限性

Logisim 不会尝试检测时序电路：如果要求它分析一个时序电路，它仍然会创建真值表和相应的布尔表达式，尽管这些结果无法准确概括电路行为。（事实上，通用地检测时序电路是可证明不可能的，因为这相当于解决停机问题。当然，可能会希望 Logisim 至少尝试寻找触发器或导线回路，但它并不会这样做。）因此，不应随意使用组合分析系统：只有在确认待分析电路确实是组合电路时才应使用它。

Logisim 可能会对原电路做出一个意想不到的更改：组合分析系统要求每个输入和输出都有唯一名称，并且名称必须符合 Java 标识符规则。（大致来说，每个字符必须是字母或数字，第一个字符必须是字母，且不允许空格。）它会尝试使用引脚已有的标签；如果没有标签，则使用默认名称。如果已有标签不符合 Java 标识符规则，Logisim 会尽可能从标签中提取一个有效名称。

顺便提一下，真值表中输入的顺序会与原电路中从上到下的顺序一致；如果纵向位置相同，则按从左到右排序。（输出的顺序同样如此。）
