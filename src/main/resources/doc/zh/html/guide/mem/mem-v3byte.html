v3.0 十六进制字节格式（普通大端）

此文件结构与先前版本相同，第一行指示结构类型，后续行表示存储器中的字节序列。它们采用 ASCII 格式，仅包含十六进制符号（两位数字），不带 0x 标记。 

v3.0 hex bytes plain big-endian
3f45a8c96ea00042f261613443f8b2cb
50950e0604427da5a9641e91526c7970
a7eb2d655343913e6b7d39db17730c77
058ea8ae931cbe211d218d412c76a495

换行符不被解释。除第一行外，没有布局规则。空格将被忽略。

可以使用“#”符号在文件中添加注释。从“#”符号开始到行末的所有字符都会被忽略。

无需在数据前指定 0x 前缀；如果存在，该前缀会被直接忽略。

如果文件的数据集长度小于存储器空间，则剩余单元对 ROM 初始化为 0；对 RAM 则根据项目选项中的参数初始化为 0 或随机值。

在大端模式下，无论存储器字的位宽是多少，存储器都按字节序列处理，从左到右依次写入高字节和低字节。上图的蓝/绿色区域把十六进制数字显示为二进制值，可以看到文件中的字节（顶部）按顺序复制到存储器中，并分别给出 4 位、7 位和 12 位字宽的示例。注意字宽不是 8 的倍数时会产生对齐偏移，例如 7 位字宽的存储器。

此格式遵循与先前格式相同的规则。区别在于读取方向：在小端模式下，从右向左读取。

v3.0 十六进制字节格式（带地址大端）

此格式与先前格式相同，但可以指定每一行数据写入的起始存储器字地址。
该地址为十六进制，后跟字符 :。 

v3.0 hex bytes addressed big-endian
00: 3f45a8c96ea75086cce5d39d87456709
20: 56788888882efff0ec45670900000000
30: 9863fec8a2d75d342e1f008090445578

在此示例中，地址 0x10 处的 8 个字未指定。这些单元对 ROM 初始化为 0；对 RAM 则根据项目选项中的参数初始化为 0 或随机值。
