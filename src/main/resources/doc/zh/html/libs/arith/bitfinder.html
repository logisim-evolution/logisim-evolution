位查找器

库： 算术
引入版本： 2.6.1
外观：

行为

该组件接收一个多位输入，并确定某一位的索引；索引从最低位开始，从 0 编号。具体查找哪个索引取决于 类型 属性，如下表中 8 位示例输入 11010100 所示。

类型 输入 11010100 时的输出
最低位1 2
最高位1 7
最低位0 0
最高位0 5

对于最低位 1，输出为 2，因为从最低位开始以 0 编号时，找到的第一个 1 位于索引 2。（索引 0 和 1 的位均为 0。）对于最高位 1，输出为 7，因为最高位的 1 位于索引 7（同样从最低位以 0 编号）。

组件南侧的输出指示是否找到了所需的位。在上述输入 11010100 的示例中，南侧输出在所有情况下均为 1。但如果输入为 00000000 且组件要查找最低位 1，则南侧输出为 0，东侧输出也为 0。

如果在搜索所需值时遇到既不是 0 也不是 1 的值（该位可能是浮空或错误值），则两个输出都会完全由错误位组成。请注意，只有在找到所需位之前遇到有问题的位时才会发生这种情况：对于输入 x1010100，如果要查找最低位 1，输出仍为 2；但如果组件类型要求查找最高位 1 或最高位 0，则会得到错误值，因为有一个错误位位于比目标位更高的位置。

引脚

西侧（输入，位宽与 数据位宽 属性一致）
要搜索所需位的多位输入。
东侧（输出，位宽按下述方式计算）
所需位的索引，从最低位以 0 编号。位宽是存储最大可能索引所需的最小位数，该索引比 数据位宽 属性的值小 1。
南侧（输出，位宽 1）
如果找到所需位则为 1；如果所有输入位均为所需位的相反值则为 0；如果在所需位之前发现非 0、非 1 的值则为错误值。

属性

当组件被选中或正在放置时，Alt-0 到 Alt-9 会更改其 数据位宽 属性。

数据位宽
输入的位宽。
类型
指示要搜索的位——最低位 0、最高位 0、最低位 1 或最高位 1。
