TCL 脚本规范

Logisim 会在一个包装器中运行 TCL 脚本。

包装器 API

包装器提供与 Questasim/Modelsim API 兼容的仿真接口。当前只支持少数函数：

examine <signal>
获取缓冲信号值。返回值为包含信号二进制值的字符串。
<signal> 为信号名称。如果其中包含信号路径（针对 Questasim），路径会被忽略（例如 /top_sim/component2/Signal_2_i 会变为 Signal_2_i）。
不支持单比特读取（例如 examine Signal_2_i(3)）。

force <signal> <value>
设置缓冲信号值。
<signal> 为信号名称。如果其中包含信号路径（针对 Questasim），路径会被忽略（例如 /top_sim/component2/Signal_2_i 会变为 Signal_2_i）。
<value> 为二进制字符串。如果字符串不完整（缺少 MSB），会用 X 补齐（例如，将 10100 设置到 8 位信号时会变为 XXX10100）。

run
向 Logisim 请求单步仿真。

包装器回调

每个仿真步骤中，包装器都会调用 refresh {} 回调。通常可以在此回调中放置读取输入、计算输出、刷新 GUI 和设置输出所需的代码。

刷新回调中不允许包含 run 函数调用。否则会产生无限循环，因为 run 函数会执行一个仿真步骤，进而再次调用刷新回调。

refresh {} {
# 使用 examine 函数读取输入
# 计算所需的值
# 刷新界面
# 使用 force 函数设置输出
}
