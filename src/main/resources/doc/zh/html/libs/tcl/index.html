TCL 脚本规范

TCL 脚本由 Logisim 在一个包装器中启动。

包装器 API

包装器提供一个与 Questasim/Modelsim API 兼容的 API 来运行仿真。仅提供少数几个函数：

examine <signal>
获取缓冲信号值。返回值为包含信号二进制值的字符串。
<signal> 信号名称。如果包含信号路径（针对 Questasim），路径将被忽略（例如 /top_sim/component2/Signal_2_i 将变为 Signal_2_i）。
不支持单比特读取（例如 examine Signal_2_i(3)）。

force <signal> <value>
设置缓冲信号值。
<signal> 信号名称。如果包含信号路径（针对 Questasim），路径将被忽略（例如 /top_sim/component2/Signal_2_i 将变为 Signal_2_i）。
<value> 二进制值的字符串。如果字符串不完整（缺少 MSB），将用 X 补齐（例如，将 10100 设置到 8 位信号时变为 XXX10100）。

run
向 Logisim 请求单步仿真。

包装器回调

在每个仿真步骤中，包装器调用 refresh {} 回调。通常可以在此包装器中放置读取输入、计算输出、刷新 GUI 和设置输出所需的代码。

刷新回调中不允许包含 run 函数调用。如果这样做，将产生无限循环，因为 run 函数会执行一个仿真步骤，进而调用刷新回调。

refresh {} {
# Read inputs with examine function
# Compute some values
# Refresh the UI
# Set the outputs with force function
}
