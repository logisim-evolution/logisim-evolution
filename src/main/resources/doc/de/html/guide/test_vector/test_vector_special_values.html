Spezielle Werte

Das Testvektor-Format unterstützt zwei spezielle Werte für flexibleres Testen:

  • <DC> (Don't Care): Wenn für einen Ausgangspin verwendet, besteht dieser Wert immer, unabhängig von der tatsächlichen Ausgabe. Dies ist nützlich, wenn Sie sich während eines Tests nicht um bestimmte Ausgaben kümmern. Groß-/Kleinschreibung wird ignoriert (<dc> funktioniert auch).
  • <float>: Wenn für einen Eingangspin verwendet, treibt dies einen schwebenden (hochohmigen) Wert an. Wenn für einen Ausgangspin verwendet, erwartet dies, dass die Ausgabe schwebend ist (UNBEKANNTER Wert). Groß-/Kleinschreibung wird ignoriert (<FLOAT> funktioniert auch).

Hier ist ein Beispiel mit speziellen Werten:

# Test with don't care and floating values
Input   Enable Output  <set> <seq>
1       1      <DC>    0     0
<float> 0      <float> 0     0

Im ersten Test ist die Ausgabe als "Don't Care" markiert, sodass jeder Ausgabewert besteht. Im zweiten Test ist der Eingang schwebend und die Ausgabe wird ebenfalls als schwebend erwartet.
