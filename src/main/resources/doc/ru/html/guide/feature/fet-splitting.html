Разветвители (Splitters)

При проектировании устройств с многоразрядными шинами данных часто возникает необходимость разделить их на отдельные проводники или группы проводников (подшины). Для решения этой коммутационной задачи используется компонент Разветвитель () из базовой библиотеки Монтаж.

Рассмотрим пример схемы, выполняющей побитовую операцию И между старшей и младшей тетрадами (полубайтами) 8-разрядного входного сигнала. В данном случае 8-разрядный сигнал поступает с входного порта и должен быть разделен на два 4-разрядных канала. Разветвитель принимает на вход 8-разрядную шину, разделяет её на два 4-разрядных ответвления, которые затем подаются на соответствующие входы логического элемента «И»:

Разветвитель является универсальным компонентом: он может использоваться как для демультиплексирования (разделения одной шины на несколько изолированных линий), так и для мультиплексирования (объединения нескольких независимых сигналов в единый многоразрядный шлейф). В среде Logisim-evolution разветвители являются пассивными двунаправленными компонентами: они способны транслировать сигналы в любом направлении, в том числе встречно и одновременно. На иллюстрации ниже показан пример топологии, где сигнал проходит слева направо через два разветвителя, затем возвращается в обратном направлении и вновь уходит направо к выходному контакту:

Для корректной настройки разветвителя необходимо оперировать правильной терминологией. В архитектуре Logisim-evolution одиночный объединённый контакт компонента называется стволом (trunk), а группа раздельных контактов с противоположной стороны — ответвлениями (branches).

Атрибуты разветвителя

  • Направление (Facing): указывает сторону, в которую направлены ответвления относительно ствола.
    Ствол (общий контакт) Ответвления
  • Разветвление (Fan Out): определяет физическое количество выходных ответвлений на корпусе компонента.
  • Разрядность входа (Bit Width In): задаёт общую логическую ширину (разрядность) ствола шины.
  • Внешний вид (Appearance): определяет визуальное позиционирование ответвлений относительно ствола. Доступны следующие варианты:
    Правосторонний Левосторонний
    По центру Классический (Legacy)
  • Интервал (Spacing): определяет физическое расстояние (в шагах координатной сетки холста) между соседними контактами ответвлений.
  • Бит x (Bit x): таблица маршрутизации. Определяет, к какому конкретному ответвлению аппаратно подключён каждый бит x общей шины. Если к одному ответвлению привязано несколько битов, их относительный порядок внутри ответвления строго соответствует их изначальному порядку на шине. Logisim-evolution аппаратно блокирует возможность подключения одного и того же бита шины к нескольким ответвлениям одновременно.
Примечание:

Изменение атрибутов Разветвление или Разрядность входа приводит к сбросу пользовательских настроек маршрутизации. В этом случае биты общей шины будут автоматически перераспределены по доступным ответвлениям максимально равномерно.
