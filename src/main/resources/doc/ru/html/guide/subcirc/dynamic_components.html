Динамические компоненты (Dynamic Components)

Динамические компоненты предоставляют возможность визуального наблюдения и интерактивного взаимодействия с элементами, скрытыми внутри подсхем, непосредственно с более высоких уровней иерархии проекта.

Суть проблемы и её решение

При стандартном проектировании, если разместить индикатор (например, светодиод) внутри схемы (допустим, MyNiceCircuit), а затем использовать её в качестве компонента на главном холсте (MainToplevel), этот индикатор окажется полностью скрыт внутри графического блока подсхемы. Для наблюдения за его текущим состоянием потребовалось бы принудительно переходить в режим отладки подсхемы.

Функционал динамических компонентов решает эту проблему: он позволяет проецировать визуальное состояние индикаторов и органов управления из внутренних схем непосредственно на условное графическое обозначение (УГО) родительского компонента.

Важное свойство:

Данный механизм работает рекурсивно. Это означает, что на графический корпус главной схемы верхнего уровня могут корректно транслироваться динамические компоненты со всех нижележащих уровней вложенности проекта.

Инструкция по настройке

  1. Откройте чертёж схемы, содержащей целевой компонент (в нашем примере — MyNiceCircuit).
  2. Перейдите в Редактор внешнего вида для этой подсхемы.
  3. На панели инструментов редактора выберите инструмент «Динамический компонент» ().
  4. В открывшемся диалоговом окне отобразится список всех поддерживаемых элементов, присутствующих на чертеже данной схемы. Выберите те, которые необходимо проецировать на внешний корпус, и разместите их в нужных местах УГО.

Поддерживаемые элементы

На данный момент механизм динамического проецирования на внешний корпус поддерживается для следующих компонентов:

Светодиод (включая RGB-светодиод)
7-сегментный индикатор
Шестнадцатеричный индикатор
Светодиодная матрица
Терминал (TTY)
Датчик (Probe)
Регистр
Счётчик
Экран VGA (SoC)
Имитационная модель процессора Nios2s / Имитационная модель процессора RISC-V (расширение IM)
