Контекстное меню и файлы

Контекстное меню

Контекстное меню для компонентов памяти содержит четыре дополнительные опции по сравнению с базовым меню остальных компонентов:

| Изменить содержимое… |
Открывает шестнадцатеричный редактор для просмотра и изменения данных памяти.
| Очистить содержимое |
Заполняет ПЗУ (ROM) нулями. Для ОЗУ (RAM) ячейки заполняются нулями или случайными значениями, в зависимости от параметров на вкладке Параметры проекта.
| Загрузить образ… |
Заменяет текущее содержимое памяти данными из выбранного файла. Формат файла обычно распознаётся автоматически; если этого не произошло, появится панель файлов памяти для ручного выбора подходящего формата.
| Сохранить образ… |
Сохраняет текущее содержимое памяти в файл, используя один из описанных ниже форматов.

Форматы файлов

  • v2.0 raw: Исторический формат Logisim, поддерживающий сжатие через кодирование повторов (RLE).
  • v3.0 hex words plain: Формат, близкий к v2.0, но данные структурированы по словам.
  • v3.0 hex words addressed: Аналогично предыдущему, но с возможностью явного указания начального адреса для каждой строки.
  • v3.0 hex bytes plain big-endian
    v3.0 hex bytes plain little-endian: Данные организованы в виде простого байтового потока.
  • v3.0 hex bytes addressed big-endian
    v3.0 hex bytes addressed little-endian: Байтовый поток с поддержкой явного указания адресов.
  • Binary data big-endian
    Binary data little-endian: Чистый (raw) двоичный образ памяти, без шестнадцатеричного текстового кодирования.
  • Ascii byte escape big-endian
    Ascii byte escape little-endian: Текстовый файл в кодировке ASCII. Непечатаемые символы представлены escape-последовательностями (как в языке C).
