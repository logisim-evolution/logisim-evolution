Logisim-evolution поддерживает базовую возможность запуска и выполнения схем непосредственно из командной строки. Это предназначено как для автоматизированной проверки цифровых проектов с помощью скриптов, так и для автоматической оценки выполненных учащимися лабораторных работ.

Рассмотрим, как запустить схему из командной строки. Для примера предположим, что мы построили схему, показанную ниже, и сохранили её в файл с именем adder-test.circ. Она использует двухбитный сумматор в качестве подсхемы и последовательно перебирает все 16 возможных входных комбинаций с помощью счётчика.

После сборки этой схемы мы запускаем Logisim-evolution из командной строки, передавая имя файла проекта и параметр -tty с аргументом table.

java -jar logisim-filename.jar adder-test.circ -tty table

Logisim-evolution загрузит схему без вывода графического интерфейса и сразу начнёт её выполнение, генерируя такты моделирования на тактовом входе с максимально возможной скоростью и производя расчёт распространения сигналов после каждого перепада. По окончании каждого процесса распространения сигналов Logisim-evolution опрашивает текущие значения на выходных контактах; если какое-либо из них изменилось по сравнению с предыдущим шагом, то в стандартный вывод выводятся все текущие значения, разделённые знаками табуляции. Если на схеме присутствует выходной контакт со специальным именем halt (чувствительным к регистру), его значение не выводится в общую таблицу, но как только значение на этом контакте становится равным 1 после очередного такта, Logisim-evolution завершает моделирование.

Для нашего примера Logisim-evolution выведет таблицу, представленную ниже. Поскольку у нас есть два выходных контакта, соответствующих входам a и b двухбитного сумматора, эти значения выводятся в первых двух столбцах. Третий столбец соответствует выходу сумматора sum. Столбцы упорядочиваются слева направо в соответствии с тем, как соответствующие им контакты расположены на схеме сверху вниз.
