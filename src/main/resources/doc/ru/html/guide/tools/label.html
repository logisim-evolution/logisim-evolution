Метка компонента (Label)

Метка — это встроенное текстовое свойство компонента схемы (например, контакта, логического элемента или триггера), служащее для его идентификации, назначения имени переменной или документирования.

Критические правила и особенности работы в Logisim-evolution:
  • Только латиница: Имя метки обязано начинаться с буквы и может содержать исключительно латинские буквы, цифры и символы подчёркивания. Использование кириллицы, пробелов или спецсимволов категорически запрещено, так как это приведёт к ошибке синтаксиса при компиляции.
  • Уникальность на схеме: Каждая метка должна быть уникальной в пределах текущего холста схемы. Недопустимо называть разные компоненты (например, два разных входных контакта) одинаковыми именами.
  • Физическая привязка: Метка жестко связана со своим компонентом. При перемещении элемента по холсту его метка автоматически перемещается вместе с ним.
  • Влияние на синтез VHDL: В отличие от простого текста, метки в Logisim-evolution играют ключевую роль. Имя метки, присвоенное входному или выходному контакту (Pin), становится официальным идентификатором соответствующего порта в автоматически генерируемом VHDL-коде.
Важное предупреждение:

Не путайте этот инструмент с текстовой надписью (аннотацией)! Метка является неотъемлемой частью компонента, задаёт имя переменной для процесса моделирования и компилятора ПЛИС и перемещается вместе с деталью. Обычная текстовая надпись (которая будет описана в следующем разделе) — это просто абстрактный комментарий на холсте для человека, который полностью игнорируется при генерации VHDL.

Атрибуты и настройка

Метка (Текст)
Строковое значение имени компонента. Его можно задать в панели свойств выбранного элемента или отредактировать прямо на холсте, кликнув по компоненту Инструментом «Текст».
Шрифт метки
Настройка гарнитуры, размера и стиля начертания (полужирный, курсив) для отображения текста метки.
Положение метки
Определяет, с какой стороны от графического корпуса компонента будет отображаться его текстовое имя (доступны варианты: вверх, вниз, вправо, влево).
Горизонтальное выравнивание
Задаёт положение текста по горизонтали относительно точки привязки метки к корпусу:
  • По левому краю: текст разворачивается вправо от точки привязки;
  • По центру: надпись центрируется строго посередине;
  • По правому краю: текст уходит влево, заканчиваясь в точке привязки.
Вертикальное выравнивание
Задаёт положение надписи по вертикали относительно точки привязки к корпусу:
  • По базовой линии: выравнивание происходит по опорной линии шрифта;
  • По верху: верхний край букв совмещается с точкой привязки;
  • По центру: текст центрируется вертикально;
  • По низу: нижний край букв совмещается с точкой привязки.

Примечание: Выравнивание по верхнему и нижнему краям рассчитывается автоматически на основе метрик конкретного шрифта. Оно остаётся стабильным и аккуратным, даже если строка физически не содержит символов с выступающими вверх (как «б») или вниз (как «у») элементами.
