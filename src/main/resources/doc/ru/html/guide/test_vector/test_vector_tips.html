Советы и рекомендации

Вот несколько советов по наиболее эффективному использованию функции тестовых векторов:

  1. Указывайте разрядность в заголовках: Обязательно указывайте разрядность каждого входного или выходного контакта в заголовке, если она больше 1 бита (например, A[8]).
  2. Используйте подчёркивания для улучшения читаемости: Вы можете использовать символы подчёркивания (_) в любом месте числовых значений (они полностью игнорируются при разборе). Примеры: 0x0000_1111, 0o1234_5670, 1111_0000, 1_234 или -5_000.
  3. Выравнивайте столбцы пробелами: Выравнивание начальных позиций значений и заголовков с помощью пробелов значительно улучшает читаемость файла. В противном случае легко перепутать, какое значение относится к какому сигналу, что затрудняет поиск причин непрохождения тестов.
  4. Применяйте последовательности для тестирования схем с внутренним состоянием: Если схема содержит элементы памяти (триггеры, регистры, оперативное запоминающее устройство), используйте последовательные тесты для проверки переходов состояний.
  5. Используйте безразличные значения для частичной проверки: При проверке сложных схем используйте специальное обозначение <DC> для выходов, правильность работы которых на данном шаге не имеет значения.
  6. Проверяйте третье состояние выходов: Используйте специальное обозначение <float> для проверки высокоимпедансного состояния (Z) на выходах схемы (например, при тестировании шинных формирователей).
  7. Группируйте тесты с помощью наборов: Используйте столбец <set> для группировки логически связанных тестов в последовательности. Тесты с одинаковым номером набора выполняются последовательно в порядке возрастания шагов <seq> с сохранением состояния схемы между ними.
  8. Смешивайте комбинационные и последовательные тесты: Вы можете свободно совмещать комбинационные тесты (шаг <seq> равен 0 или отсутствует) и последовательные шаги в пределах одного файла.

Обратная совместимость

Все созданные ранее файлы тестовых векторов продолжают работать без каких-либо изменений. Новые функции являются полностью опциональными:

  • Если столбцы <set> и <seq> отсутствуют, все тесты выполняются как комбинационные (поведение по умолчанию).
  • Если специальные значения <DC> и <float> не используются, применяется стандартное сравнение точных значений.
  • Оригинальный формат файлов классического Logisim поддерживается в полном объёме.
