Поведение

Компонент Контакт служит интерфейсом ввода или вывода для схемы. Его роль определяется атрибутом Тип контакта.

Внешний вид (задаётся атрибутом Внешний вид):

  • Стиль «Стрелка» (Arrow): входные контакты имеют заострённую форму со стороны подключения, а выходные — прямоугольную.
  • Стиль «Классический» (Classic): входные контакты отрисовываются как квадраты или прямоугольники, а выходные — как круги или скруглённые прямоугольники.

Примечание: Отдельные биты передаваемого значения всегда отображаются внутри самого компонента (за исключением режима печати, где выводится только разрядность).

Использование в иерархии: Контакт — удобный элемент для интерактивного управления схемой на начальном этапе. Однако его главная функция раскрывается при создании иерархических проектов (см. раздел Подсхемы Руководства пользователя). Контакты, размещённые на чертеже текущей схемы, автоматически становятся внешними контактами (портами) на её графическом обозначении при вставке в другую схему.

Контакты

Компонент имеет единственный физический контакт на холсте:

  • Он работает как вход для самого компонента, если тот настроен как выходной порт схемы.
  • Он работает как выход (источник сигнала), если компонент настроен как входной порт схемы.

Разрядность этого контакта соответствует атрибуту Биты данных, а положение на корпусе — атрибуту Направление.

Атрибуты

Горячие клавиши при выборе или добавлении компонента:

  • Комбинации от Alt-0 до Alt-9 — изменяют Биты данных. (Клавиши можно нажимать последовательно: например, Alt-1 затем Alt-2 задаст разрядность 12);
  • Клавиши со стрелками — изменяют Направление (ориентацию компонента);
  • Alt + Клавиши со стрелками — изменяют Направление метки.
Направление
Сторона компонента, на которой располагается его линия связи (электрический контакт).
Тип контакта
Определяет, является ли компонент входным или выходным портом текущей схемы.
Биты данных
Разрядность значения, принимаемого или выдаваемого контактом.
Поведение
(Доступно только для входных контактов). Определяет обработку неопределённых (высокоимпедансных) состояний U (или Z):
  • Два состояния (0/1) или Три состояния (0/1/Z): состояния U передаются в схему «как есть».
  • Подтяжка вверх (Pull Up): заменяет любые состояния U на логическую 1 перед отправкой в схему.
  • Подтяжка вниз (Pull Down): заменяет состояния U на логический 0.
Данный атрибут также контролирует ручной ввод: только режим «Три состояния» позволяет пользователю интерактивно вводить неопределённые биты U через интерфейс.
Метка
Текстовое название компонента.
Направление метки
Положение текста метки относительно корпуса компонента.
Шрифт метки
Выбор стиля шрифта для метки.
Система счисления
Определяет формат отображения многобитных значений на корпусе компонента. Пример отображения двоичного значения 1010:
  • Двоичная: 1010b
  • Восьмеричная: 12o
  • Знаковая десятичная: -6s
  • Беззнаковая десятичная: 10u
  • Шестнадцатеричная: Ah
  • С плавающей точкой (16 бит): 6.0E-07f
  • С плавающей точкой (32 бита): 1.4E-44f
  • С плавающей точкой (64 бита): 4.9E-323f
  • С плавающей точкой (иная разрядность): NaNf
Внешний вид
Выбор графического стиля компонента («Стрелка» или «Классический»).
Значение при сбросе
(Только для входных контактов). Позволяет задать стартовое шестнадцатеричное значение, которое автоматически применится при сбросе моделирования. Если контакт работает в режиме «Три состояния», при сбросе все биты принудительно перейдут в неопределённое состояние (U).

Поведение Инструмента «Нажатие»

  • На выходном контакте: Действие отсутствует (инструмент только выводит атрибуты контакта на панель свойств).
  • На входном контакте: Инвертирует значение бита, по которому произведён щелчок.
    • В режиме «Три состояния» клик циклически переключает бит по кругу: 0 → 1 → U.
    • В системе счисления «С плавающей точкой» клик открывает диалоговое окно для текстового ввода точного значения.
  • При отладке подсхемы: Если вы просматриваете внутреннее состояние подсхемы (см. «Отладка подсхемы»), значения на её портах жёстко задаются родительской схемой. Попытка изменить их вручную приведёт к конфликту; Logisim выдаст предупреждение и предложит разорвать синхронизацию состояний с родительской схемой.
