Поведение

Счётчик хранит одно значение, выдаваемое на выход Q. Значение обновляется по тактовому сигналу (треугольник на южной стороне корпуса) на основе состояний входов Загрузка (Load), Разрешение (ct) и Направление счёта (UD):

Загрузка Разрешение (ct) Направление (UD) Действие по такту
не 1 0 любое Значение счётчика остаётся неизменным.
не 1 не 0 не 0 Значение счётчика увеличивается.
не 1 не 0 0 Значение счётчика уменьшается.
1 любое любое Счётчик загружает значение со входа D.

Диапазон счёта настраивается с помощью атрибута Максимальное значение. Когда счётчик достигает этого максимума, следующее увеличение (инкремент) возвращает его значение к 0. Если же значение счётчика равно 0, то уменьшение (декремент) возвращает его к максимальному значению.

В дополнение к выходу Q компонент имеет однобитный выход перенос. На этом выходе устанавливается 1, когда счётчик достигает максимального значения и входы указывают на увеличение (инкремент) значения на следующем такте, либо когда значение счётчика равно 0 и входы указывают на уменьшение (декремент) на следующем такте. Сигналы на входах Загрузка (Load) и Разрешение (ct) не влияют на этот выход напрямую.

Вход асинхронного сброса Очистка (Reset) сбрасывает значение счётчика в 0 (все нули). Пока на этом входе удерживается уровень 1, значение счётчика зафиксировано на 0 независимо от тактового входа.

Контакты

Ниже приведено описание контактов, которое относится как к классическому внешнему виду, так и к внешнему виду в стиле Logisim-evolution (который использует стандартные графические обозначения IEC/IEEE).

Правая сторона, отмечен Q (выход, разрядность соответствует атрибуту Биты данных):
Выдаёт значение, хранящееся в данный момент в счётчике.
Правая сторона, нижний контакт (выход, разрядность равна 1):
Перенос (показан как 3CT=MAX / 4CT=0): на этом выходе устанавливается 1, когда счётчик достигает своего максимального значения и считает вверх, либо когда он равен 0 и считает вниз.
Левая сторона, верхний контакт (вход, разрядность равна 1):
Загрузка (показана как M1): когда на этом входе уровень 1, счётчик загружает значение со входа D по следующему активному фронту тактового сигнала независимо от состояния входа Разрешение (ct).
Левая сторона, средний контакт, отмечен D (вход, разрядность соответствует атрибуту Биты данных):
Данные: значение, которое будет загружено в счётчик, когда активен вход Загрузка.
Левая сторона, нижний контакт, отмечен ct (вход, разрядность равна 1):
Разрешение (ct) (показано как G5): при любом значении, отличном от 0, счётчик инкрементирует или декрементирует своё значение по тактовому сигналу (при условии, что на входе Загрузка не уровень 1).
Верхняя сторона, отмечен UD (вход, разрядность равна 1):
Направление счёта (показан как M3/M4): при любом значении, отличном от 0, счётчик считает на увеличение (вверх). При значении 0 счётчик считает на уменьшение (вниз). Этот вход определяет направление счёта, когда вход Загрузка (Load) не равен 1, а вход Разрешение (ct) не равен 0.
Нижняя сторона, отмечен треугольником (вход, разрядность равна 1):
Тактовый вход (показан как C6 / 2,3,5+ / 2,4,5-): тактовый сигнал. Срабатывание происходит в момент, определяемый атрибутом Срабатывание. В обозначении стандарта Evolution несколько меток отражают внутренние зависимости от режимов счёта/загрузки.
Нижняя сторона, отмечен 0 или R (вход, разрядность равна 1):
Сброс (показан как R): пока на этом входе уровень 1, значение счётчика асинхронно удерживается равным 0.

Внешний вид Logisim-Evolution (стандарт IEEE/IEC)

Когда атрибут Внешний вид установлен в Logisim-Evolution, счётчик использует стандартные графические обозначения IEC/IEEE (IEC 617-12 и IEEE Std 91-1984). Этот вариант внешнего вида включает дополнительные управляющие контакты и индикаторы режимов:

Обозначение Метка Функция
G (или G5) Разрешение счёта Когда на входе высокий уровень (1), счёт разрешён. Когда низкий (0), счёт приостанавливается независимо от тактовых импульсов.
M1 [load] Управление режимом загрузки. Активирует загрузку со входа данных (Data).
M2 [count] Управление режимом счёта. Активирует операцию счёта.
M3 [up] Управление счётом на увеличение. Выбирает режим инкремента.
M4 [down] Управление счётом на уменьшение. Выбирает режим декремента.
2,3,5+/C6 Clock+ Тактовый вход для определённых комбинаций режимов (при активном разрешении счёта).
2,4,5- Clock- Альтернативный тактовый вход для различных режимов работы.

Примечание: Во внешнем виде Logisim-Evolution могут отображаться дублирующие тактовые контакты и контакты переноса. Они представляют различные режимы работы счётчика в соответствии со стандартом IEC/IEEE. Функциональность идентична классическому виду; отличается только графическое представление.

Атрибуты

Когда компонент выбран или добавляется на холст, горячие клавиши от Alt-0 до Alt-9 изменяют его атрибут Биты данных.

Биты данных
Разрядность значения, хранящегося в счётчике и выдаваемого на выход.
Максимальное значение
Максимальное значение диапазона счёта, при достижении которого счётчик устанавливает выход переноса.
Действие при переполнении
Поведение при попытке увеличить значение счётчика выше максимального или уменьшить его ниже 0. Поддерживаются четыре варианта:
Закольцовывать
При увеличении следующее значение становится равным 0, при уменьшении — максимальному значению.
Оставаться на значении
Значение счётчика фиксируется на максимуме (или на 0 при уменьшении).
Продолжать счёт
Счётчик продолжает счёт в рамках разрядности, указанной в атрибуте Биты данных.
Загрузить следующее значение
Следующее значение загружается со входа D.
Срабатывание
Определяет реакцию на тактовый сигнал: Фронт (переход с 0 на 1) или Спад (переход с 1 на 0).
Метка
Текстовое название или обозначение элемента на схеме.
Шрифт метки
Гарнитура, размер и начертание шрифта для отображения метки.
Внешний вид
Определяет визуальное представление компонента: «Классический Logisim» или «Logisim-Evolution» (символика IEC/IEEE).

Поведение Инструмента «Нажатие»

Нажатие на счётчике переводит фокус ввода клавиатуры на компонент (выделяется красной рамкой), после чего ввод шестнадцатеричных цифр позволяет изменить значение, хранящееся в счётчике.
