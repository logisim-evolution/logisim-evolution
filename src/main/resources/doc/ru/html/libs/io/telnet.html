Поведение

Компонент Telnet открывает серверный TCP-сокет на настроенном порту. Внешняя программа может подключиться к этому порту с помощью telnet, netcat, браузера или любого другого TCP-клиента. Байты, полученные от клиентов, сохраняются во внутреннем буфере. Байты, отправленные схемой, записываются последнему подключенному клиенту.

Компонент передает по одному байту за раз. Выход доступности данных av равен 1, пока буфер приема содержит хотя бы один байт. Когда на вход чтения rd подается 1, самый старый полученный байт появляется на выходе out (при наличии данных). По настроенному активному фронту тактового сигнала байт удаляется из приемного буфера, если на rd подана 1, а байт на входе in отправляется клиенту, если на входе разрешения записи wr подана 1.

If ни один клиент не подключен, операции записи не имеют видимого эффекта. Если несколько компонентов Telnet используют один и тот же порт, они делят между собой один и тот же серверный сокет; обычно в проекте следует использовать не более одного компонента Telnet для каждого TCP-порта.

Контакты:

Левая сторона, верхний контакт с меткой in (вход, разрядность равна 8):
Байт для отправки подключенному TCP-клиенту.
Правая сторона, верхний контакт с меткой out (выход, разрядность равна 8):
Самый старый байт, полученный от TCP-клиента, пока на rd подана 1 и выход av равен 1. Состояние выхода неинициализировано (U), пока rd равен 0.
Левая сторона, средний контакт, отмеченный треугольником (вход, разрядность равна 1):
Тактовый сигнал — по активному фронту тактового сигнала выполняются разрешенные операции чтения и записи.
Левая сторона, нижний контакт с меткой wr (вход, разрядность равна 1):
Разрешение записи: при 1 активный фронт тактового сигнала отправляет байт со входа in подключенному клиенту.
Правая сторона, нижний контакт с меткой rd (вход, разрядность равна 1):
Разрешение чтения: при 1 самый старый полученный байт выдается на выход out, а по тактовому сигналу этот байт удаляется из буфера приема.
Правая сторона, средний контакт с меткой av (выход, разрядность равна 1):
Доступность данных: равен 1, если в буфере приема находится как минимум один байт.

Атрибуты

Использовать протокол Telnet
Когда включено, сервер выполняет базовое согласование параметров Telnet и игнорирует последовательности команд Telnet от клиента. Оставьте это значение отключенным для «сырых» TCP-клиентов (raw TCP), таких как netcat, простые скрипты и большинство примеров для браузеров.
Порт
TCP-порт, на котором компонент ожидает входящие соединения. По умолчанию — 8000.
Длина буфера
Количество полученных байтов, которые могут ожидать в очереди во внутреннем буфере.
Срабатывание
Определяет, выполняются ли разрешенные операции чтения и записи по фронту или по спаду тактового сигнала.
Метка, Положение метки, Шрифт метки
Настройка дополнительной текстовой метки компонента.
