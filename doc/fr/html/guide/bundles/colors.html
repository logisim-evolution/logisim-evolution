Couleurs du câblage

Nous sommes maintenant en mesure de résumer l'ensemble des couleurs que Logisim peut utiliser pour les câbles. Le circuit suivant les illustre toutes.

  • Gris: La largeur en bit du câblage est inconnue. Ceci arrive quand le câble n'est attaché à aucune entrée ni sortie de composant.

  • Bleu: Le câble transporte une valeur à un bit, mais rien ne force une valeur spécifique sur le câble. Nous appelons ceci une un bit flottant; certains l'appellent une valeur de haute impédance. Dans cet exemple, le composant qui envoie une valeur sur le câble est un bit à trois états, il peut donc émettre cette valeur flottante.

  • Vert foncé: Le câble transporte une valeur à un bit de 0.

  • Vert clair: Le câble transporte une valeur à un bit de 1.

  • Noir: Le câble transporte une valeur de plusieurs bits. Certains ou tous les bits peuvent être indéterminés.

  • Rouge: Le câble transporte une valeur d'erreur. Ceci arrive souvent parce qu'une porte logique ne peut pas déterminer correctement sa sortie, peut-être parce qu'elle ne reçoit aucune entrée. Ceci peut aussi arriver parce que deux composants essayent d'envoyer deux valeurs différentes sur le câble. Ceci arrive dans l'exemple ci-dessus parce que l'un des bits de sortie envoie 0 sur le câble et l'autre envoie 1 sur le même câble, ce qui cause un conflit. Les câbles multibit deviennent rouges quand l'un des bits transporte une valeur d'erreur.

  • Orange: Les composants reliés au câble n'ont pas la même largeur de bits. Un câble orange est effectivement "cassé": il ne transporte aucune valeur entre les composants. Ici nous avons relié un composant à deux bits à un composant à un bit, ils sont donc incompatibles.

  • Vert-Fuchsia: Cet arrangement de couleurs indique que le fil a été sélectionné avec l'outil "poussé" (). Cela affichera l'état de la ligne dans une petite info-bulle et cela permet aussi de suivre une connexion dans un schema complexe.
